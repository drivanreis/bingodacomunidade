## ⚠️ PROBLEMAS ENCONTRADOS

### Problema 1: Scripts Interativos Bloqueavam Execução
- **Situação**: `limpa.sh` solicitava confirmação interativa ("digite 'sim'")
- **Impacto**: TPIC ficava preso esperando entrada, expirava no timeout
- **Solução Aplicada**: 
  - Usar `echo 'sim' | bash limpa.sh` para automação
  - Atualizer Phase1 para aguardar serviços realmente, não apenas timeout

### Problema 2: Run_Script Quebrando com Pipes
- **Situação**: Função não extraía corretamente script_name quando comando tinha pipes
- **Exemplo**: `echo 'sim' | bash limpa.sh` → script_name ficava "echo"
- **Solução Aplicada**: 
  - Regex para procurar por `.sh` no comando
  - Fallback para split()[0] se nenhum .sh encontrado

### Problema 3: Execução Não Era Sequencial
- **Situação**: Tentativas anteriores de forçar execução sem aguardar scripts terminarem
- **Insight do Usuário**: 
  > "Tem que ser como eu! Executar um por vez, aguardar terminar, depois próximo"
- **Solução**: Reescrever Phase1 com fluxo completamente sequencial

## 🔍 ANÁLISE DOS PROBLEMAS

### Problema A: Frontend Instability
- Frontend inicia rapidamente mas pode estar em "building" mode
- Vite (dev server) pode estar ainda compilando quando browser tenta acessar
- **Solução sugerida**: Aumentar aguarde no TPIC para >30s antes de abrir browser

### Problema B: Browser Fecha Prematuramente  
- "Target page, context or browser has been closed" indica que o browser foi fechado
- Possível: Timeout curto, browser crash, ou memória insuficiente
- **Solução sugerida**: Adicionar tratamento de reconexão

### Problema C: Intents Não Encontradas
- Intent "admin_site" não encontrada na homepage
- Intent "paroquias" não encontrada após login
- **Possível causa**: Frontend em build mode, página JavaScript ainda carregando
- **Solução sugerida**: Aguardar window.location estar pronto antes de procurar elementos
