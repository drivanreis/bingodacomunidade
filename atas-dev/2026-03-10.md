## 🎯 ASSUNTO PRINCIPAL

---

## ⚠️ PROBLEMA IDENTIFICADO

O usuário identificou **3 flaws críticos** no design inicial do TPIC:

### **Flaw 1: ROUTES dict Incompleto**
- 📍 Situação: Apenas 6 rotas em vez de 30+
- 🎯 Impacto: Cobertura de testes incompleta
- ✅ Solução: Extrair todas as 37 rotas de App.tsx
- 📊 Resultado: ROUTES dict expandido de 6 → 37 rotas

### **Flaw 2: Navegação Usando goto() (Não-realista)**
- 📍 Situação: Código usava `page.goto()` para navegar direto para URLs
- 🎯 Impacto: Testes não eram realistas - usuário real clica botões, não digita URLs
- 💬 Frase-chave: **"Você tem que se passar por mim"** (passando pela UI)
- ✅ Solução: Usar `page.click()` em vez de `page.goto()`
- 📊 Resultado: Navegação 100% por cliques

### **Flaw 3: Descobrimento de Seletores Era Estático** ⭐ CRÍTICO
- 📍 Situação: Seletores CSS hardcoded em listas
- 💭 Insight do Usuário: 
  > "O site está em desenvolvimento, botões podem sumir, surgir novos, rotas aparecem. O descobrimento tem que ser uma coisa **ATIVA, toda vez que a página for carregada**"
- 🎯 Impacto: Testes quebram toda vez que UI muda
- ✅ Solução: **Descoberta dinâmica automática** com 6 estratégias
- 📊 Resultado: Testes resilientes e adaptativos

## 🏛️ DECISÕES ARQUITETURAIS

### Paradigma: Intent-Based Dynamic Discovery ✨

```
ANTES (Hardcoded - Frágil):
selector = "button:has-text('Paroquias')"

DEPOIS (Dinâmico - Robusto):
finder = DynamicSelectorFinder(page)
selector = await finder.find_by_intent("paroquias")
```

### Decisões Tomadas:

✅ **Novo Módulo: `element_discovery.py`**
- `DynamicSelectorFinder` class com 6 estratégias de busca
- `find_by_intent()` method → procura por intenção, não seletor fixo
- Cache de seletores (performance)
- Logging detalhado

✅ **Reescrever Todas as 5 Fases**
- Phase 1: Scripts shell (sem mudanças)
- Phases 2-5: Todos com `DynamicSelectorFinder`
- Padrão único: `finder.find_by_intent("objetivo")`

✅ **6 Estratégias de Descoberta** (em ordem de confiabilidade)
1. Texto exato (várias capitalizações)
2. Texto parcial
3. Aria-labels (acessibilidade)
4. Títulos/tooltips
5. Padrões de classe CSS
6. Padrões de URL (href)

## 🔄 PADRÃO DE NAVEGAÇÃO ANTES vs DEPOIS

### ❌ ANTES (Hardcoded - Frágil)
```python
selector = "button:has-text('Paroquias')"  # Quebra se mudar!
await page.click(selector)
```

Se desenvolvedor muda para:
```html
<button aria-label="manage-parishes">Dioceses</button>
```
→ **TESTE FALHA** ❌

### ✅ DEPOIS (Dinâmico - Robusto)
```python
finder = DynamicSelectorFinder(page)
selector = await finder.find_by_intent("paroquias")  # Adapta!
```

6 estratégias tentadas automaticamente:
1. Texto exato: `"paroquias"` → Não encontra
2. Texto parcial: Procura substring → Não encontra
3. Aria-label: `aria-label*='paroquias'` → **ENCONTRA!** ✅
4. Title/Classes/Href → Não precisam ser testadas

→ **TESTE PASSA** ✅
