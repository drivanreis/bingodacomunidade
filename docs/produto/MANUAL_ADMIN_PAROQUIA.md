# MANUAL DE PRODUTO — ADMIN PARÓQUIA

## 1) Papel no sistema
O Admin Paróquia é o operador local. Ele conduz a operação da comunidade dentro dos limites da própria paróquia.

## 2) Responsabilidades principais
- Gerenciar usuários comuns da própria paróquia.
- Ativar/desativar contas e aplicar bloqueios quando necessário.
- Conduzir a operação de bingo local (quando módulo de bingo estiver habilitado).
- Apoiar recuperação de acesso dos fiéis em casos administrativos.
- Acompanhar indicadores e demandas operacionais locais.

## 3) O que pode fazer
- Acessar dashboard administrativo paroquial.
- Administrar usuários vinculados à sua paróquia.
- Criar subordinados administrativos paroquiais (se a política atual permitir).
- Operar jogos/sorteios da paróquia conforme permissões ativas.

## 4) O que não pode fazer
- Não pode executar governança global do sistema.
- Não pode criar Admin Site.
- Não pode atuar fora da sua paróquia.
- Não pode participar como fiel em jogos.

## 5) Fluxo operacional recomendado
1. Login de Admin Paróquia.
2. Revisão diária de usuários (ativos/inativos/bloqueios).
3. Planejamento e gestão de eventos de bingo.
4. Acompanhamento de suporte local (cadastro, acesso, permissões).
5. Escalonamento para Admin Site ou desenvolvedor quando sair do escopo local.

## 6) Triage de incidentes
- Conta desativada/falta de permissão local: Admin Paróquia resolve.
- Erro interno (500), CORS, falha técnica: encaminhar ao desenvolvedor.
- Regras globais de acesso/admins: escalar para Admin Site.

## 7) Regras imutáveis aplicadas
- Sem autocadastro para administradores.
- Hierarquia e rastreabilidade de criação de contas.
- Administração estritamente vinculada ao escopo paroquial.
