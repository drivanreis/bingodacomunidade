# MANUAL DE PRODUTO — ADMIN SITE

## 1) Papel no sistema
O Admin Site é o guardião da plataforma. Ele governa acesso administrativo e integridade operacional, sem participar dos jogos.

## 2) Responsabilidades principais
- Conduzir o primeiro acesso administrativo (troca de bastão do seed).
- Criar Admin Site sucessor (quando necessário).
- Criar Admin Paróquia para operação local.
- Ativar/desativar contas administrativas conforme hierarquia.
- Monitorar status global do sistema e manutenção pública.
- Acompanhar auditoria e feedbacks institucionais.

## 3) O que pode fazer
- Entrar em rotas de governança do Admin Site.
- Gerenciar paróquias e administradores paroquiais.
- Gerenciar configurações sistêmicas.
- Consultar relatórios administrativos.

## 4) O que não pode fazer
- Não pode comprar cartela.
- Não pode participar como fiel.
- Não deve operar tarefas pastorais locais de rotina (isso é do Admin Paróquia).

## 5) Fluxo operacional recomendado
1. Login administrativo do site.
2. Validar se já existe Admin Paróquia ativo.
3. Se não existir, criar o primeiro Admin Paróquia real.
4. Confirmar liberação do público (fim da manutenção inicial).
5. Revisar acessos, status de usuários e logs.

## 6) Checklist de decisão rápida
- Problema de permissão global? Admin Site.
- Necessidade de novo Admin Paróquia? Admin Site.
- Erro técnico de código/infra? Desenvolvedor.
- Gestão de fiel local? Admin Paróquia.

## 7) Regras imutáveis aplicadas
- Seed administrativo não é deletado (apenas inativado).
- Hierarquia de criação deve ser sempre respeitada.
- Admin Site não entra em fluxo de jogador.
