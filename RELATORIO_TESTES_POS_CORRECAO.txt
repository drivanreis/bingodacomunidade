TESTES APÓS REFACTORING - PROBLEMA DESCOBERTO
═══════════════════════════════════════════════════════════════════════════════

Ao refatorar useAuth para arquivo separado (src/contexts/useAuth.ts), os testes
começaram a falhar porque os mocks de AuthContext não exportavam AuthContext.

Status inicial de FALHA:
  ❌ Test Files:    7 failed | 22 passed (29 total)
  ❌ Tests:         31 failed | 231 passed (262 total)
  ❌ Erro:          "useAuth deve ser usado dentro de um AuthProvider"

Raiz do problema:
  - src/contexts/useAuth.ts importa AuthContext e chama useContext(AuthContext)
  - Os testes mockavam useAuth mas não mockavam AuthContext
  - Sem AuthContext no mock, useContext retornava undefined
  - useAuth lançava erro ao tentar usar contexto undefined

════════════════════════════════════════════════════════════════════════════════
SOLUÇÃO IMPLEMENTADA
════════════════════════════════════════════════════════════════════════════════

Estratégia: Mockar AMBOS AuthContext e useAuth.ts

ANTES (não funcionava):
  vi.mock('../../contexts/AuthContext', () => ({
    useAuth: () => ({...})  // Apenas useAuth exportado
  }));

DEPOIS (funciona):
  vi.mock('../../contexts/AuthContext', () => ({
    AuthContext: {
      Provider: ({ children }) => children,
    },
    useAuth: () => ({...})  // AuthContext agora exportado também
  }));

  vi.mock('../../contexts/useAuth', () => ({
    useAuth: () => ({...})  // Mock separado do arquivo useAuth.ts
  }));

Implementação: 7 arquivos de teste atualizados
